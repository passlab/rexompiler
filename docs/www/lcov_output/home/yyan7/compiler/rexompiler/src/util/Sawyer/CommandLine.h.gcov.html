      42             :  * @section sawyer_commandline_defns Definitions
      43             :  *
      44             :  *  @li A <em>program command line</em> is the vector of strings passed to a program by the operating system or runtime.
      45             :  *  @li A <em>commmand line argument</em> is one element of the program command line vector.
      46             :  *  @li A <em>switch</em> is a named command line argument, usually introduced with a special character sequence followed
      47             :  *      by a name, such as <code>-\-color</code>. The "--" is the <em>switch prefix</em>, "color" is the <em>switch name</em>,
      48             :  *      and "--color" is the <em>switch string</em>.
      49             :  *  @li A <em>switch argument</em> is an optional value specified on the program command line and associated with a switch,
      50             :  *      such as the word "grey" in <code>-\-color=grey</code> or <code>-\-color grey</code> (as two command-line arguments).
      51             :  *  @li A <em>switch value</em> is a switch argument that has been converted to a value within a program, such as the
      52             :  *      enumeration constant <code>GRAY</code>, whether the string on the command line was "grey" or "gray".
      53             :  *  @li A <em>non-switch</em> is a program argument that doesn't appear to be a switch. Another name for the same thing
      54             :  *      is <em>positional program argument</em>.
      55             :  *
      56             :  * @section sawyer_commandline_parts The major parts of the API
      57             :  *
      58             :  *  Program command-line parsing consists of the following major components:
      59             :  *
      60             :  *  @li The Switch class declares a switch and specifies such things as the switch name and its arguments.
      61             :  *  @li The SwitchGroup class groups related switches into collections.
      62             :  *  @li The Parser class parses a program command line to produce a ParserResult.
      63             :  *  @li The ParserResult class holds information about how a program command line was parsed by storing, among other things,
      64             :  *      a list of ParsedValue objects.
      65             :  *  @li The ParsedValue class holds details about each value parsed from a program command line, including the value's
      66             :  *      string from the command line and information about the associated switch.

      69             :  *
      70             :  *  The library is used in three phases: first, the command line is described in terms of switches and their arguments; then a
      71             :  *  parser is constructed and applied to the command line to obtain a result; and finally, the result is used by querying or
      72             :  *  pushing results into variables.

      74             :  *  Some of our goals in designing this library were influenced by other libraries. We wanted to take the good ideas of others
      75             :  *  but avoid the same pitfalls.  Our goals are:
      76             :  *
      77             :  *  @li Keep the switch declaration API as simple and terse as reasonably possible without the loss of self-documenting
      78             :  *      code.  Our approach is to use small, optional property-setting functions that can be chained together rather than
      79             :  *      excessive function overloading or magical strings.
      80             :  *  @li The switch declaration API should be able to describe all the situations that routinely occur in command-line parsing
      81             :  *      including things like optional arguments and multiple arguments, various ways to handle multiple occurrences of the
      82             :  *      same switch, short and long switch names, various switch prefixes and value separators, nestling of short switches and
      83             :  *      their arguments, standard switch actions, etc.  We accomplish this by using a consistent API of switch properties.
      84             :  *  @li Parsing of the command line should not only recognize the syntax of the command line, but also parse switch arguments
      85             :  *      into a form that is easy to use within the program.  E.g., a string like "1234" should become an <code>int</code> in
      86             :  *      the program.
      87             :  *  @li Provide an extension mechanism that is easy for beginning programmers to grasp. One way we accomplish this is by
      88             :  *      avoiding excessive use of templates for generic programming and instead rely on polymorphic classes and smart
      89             :  *      pointers, both of which are common in other object oriented languages.  Parsing command-line arguments is not typically
      90             :  *      a performance bottleneck, and when we were faced with the alternatives of performance versus features we chose
      91             :  *      features.
      92             :  *  @li The library should provide an API that is suitable for both library writers and application programmers.  In other
      93             :  *      words, a library that expects to be configured from the command line should provide a description of its command-line
      94             :  *      in a way that allows the application to augment it with its own switches, even to the extent that the application can

      96             :  *  @li The parser results should have a rich query API since this is the point at which it primarily interfaces with the

      98             :  *      push capabilities for the results: the user can query results or the library can write them directly into user-supplied

     100             :  *  @li Documentation should appear next to the thing it documents, and it should not be necessary to have more than one
     101             :  *      source-level copy of documentation.  The library is able to generate complete Unix manual pages as plain text or
