     244             :  * @section diagnostics_partial Partial messages
     245             :  *
     246             :  *  A partial message is any message for which the linefeed, "\n", hasn't been inserted.  If a message stream is unbuffered and
     247             :  *  another message is emitted to a different stream and ends up going to the same ultimate destination (e.g., standard error),
     248             :  *  then any previous partial message is interrupted, usually by appending "...", and re-emitted later when more text is added.
     249             :  *
     250             :  *  Since each message stream creates messages sequentially in an incremental manner, we sometimes want to "save" a partial
     251             :  *  message, allow other messages to be emitted from the same stream, and then complete the original message later.  This most
     252             :  *  often happens when one function creates a partial message, and then calls another function that might use the same stream
     253             :  *  before the partial message is completed.  The best way to accomplish this is by creating a new, temporary message stream
     254             :  *  and moving any partial messages to the new stream.  The copy constructor can be used, like the example above, but even more
     255             :  *  conveniently, if the source stream in the copy constructor has an outstanding partial message then that partial message is
     256             :  *  moved (not copied) to the newly constructed stream.  This allows a partial message to be emitted in the same statement that
     257             :  *  constructs the local stream:
     258             :  *
     259             :  * @code
     260             :  *  Stream m1(mlog[INFO] <<"loading \"" <<filename <<"\""); //note no "\n"
     261             :  *  Sawyer::Stopwatch timer;
     262             :  *  do_other_stuff_that_might_emit_info_messages();
     263             :  *  m1 <<"; took " <<timer <<"\n"; //original message completed now
     264             :  *  // you may continue to use m1 for additional messages...
     265             :  * @endcode
     266             :  *
     267             :  *  The documentation for @ref Sawyer::Message has additional hints and examples.
