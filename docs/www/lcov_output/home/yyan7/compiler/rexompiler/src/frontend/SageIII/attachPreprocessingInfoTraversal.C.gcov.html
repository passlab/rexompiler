      24             : The tree traversal works as follows: whenever it hits a located node
      25             : (currently: a statement), it checks if there is preprocessing info the
      26             : line number of which is less or equal than the line number of the current
      27             : located node (currently: of the current statement). If this is the case,
      28             : the corresponding preprocessing info is attached to the current

      31             : derived tree traversal class.
      32             :

      34             : preprocessingInfo objects as soon as the traversal returns to a SgFile
      35             : object and attaches trailing preprocessing information to the last located
      36             : node (currently to the last statement) that has been visited in
      37             : the file (unparse flag: "after").
      38             : 
      39             : Node that the preprocessingInfo objects are always attached to AST nodes.

      41             : you only change the mechanism which the unparser is based on!
      42             : If USE_OLD_MECHANISM_OF_HANDLING_PREPROCESSING_INFO is set to 1, then
      43             : the unparser simply ignores the preprocessingInfo objects that have
      44             : been attached to the AST nodes.
